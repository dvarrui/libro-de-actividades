# Libro de actividades

Proyecto para compartir las actividades/ejercicios/prácticas como si fueran componentes reutilizables, útiles para las clases de los **Ciclos Formativos de Informática**.

* [¿Cómo accedo a los documentos?](./docs/acceso.md)
* [¿Cómo puedo contribuir al proyecto?](./docs/contribuciones.md)
* [Proyectos similares](./docs/proyectos-similares.md)

# 1. Organización

El directorio [actividades](./actividades) es la carpeta principal. En este directorio se guardan las actividades agrupadas por áreas temáticas.

* Cada agrupación piede estar asociada con un módulo o varios o ninguno. Todo depende del punto de vista de cada profesor.
* Este catálogo no presupone ni asume ninguna programación didáctica concreta. Cada profesor define su programación y en funcion de su diseño puede ir reutilizando unas actividades u otras de su elección para completar las unidades de trabajo.
* Es posible que para diseñar una asignatura al gusto del profesor, se deban coger actividades de un área temática u otra.
* Cada fichero `.md` es un docuemento de texto simple en formato Markdown que puede enlazar, si fuera necesario, a otros documentos, imágenes o páginas web.

| Categoría                  | Uso recomendado           |
| -------------------------- | ------------------------- |
| [hardware](./hardware)     | Actividades para usar en módulos relacionados con hardware. Por ejemplo, "Fundamentos de Hardware" de 1ASIR y similares |
| [sistemas.2](./sistemas.2) | Actividades de sistemas operativos con grado de dificultad medio. Recomensado para módulos como "Implantación de Sistemas Operativos" de 1ASIR y similares|
| [sistemas.3](./sistemas.3) | Actividades de sistemas operativos con grado de dificultad avanzado. Recomendado para módulos como "Administración de Sistemas Operativos" de 2ASIR y similares |
| [global](./global)         | Contiene información para todas las actividades y que depende de la instalación específica de  mi centro |

# 2. La actividad es lo principal

El proyecto **Libro de actividades no está pensado para ser un contenedor de apuntes**. Todos los contenidos para los apuntes los tenemos ya en internet. Lo que se hace es crear actividades/ejercicios/proyectos y dentro de éstos poner enlaces a los contenidos que se van necesitando. El centro principal de los componentes reutiliables son las actividades, y los contenidos/apuntes son un complemento secundarios cuya función es ayudar a completar la actividad.

Pero... hay ocasiones en las que los contenidos de internet no están bien explicados, o están repartidos entre muchas páginas, etc. En esos casos intentamos poner en la actividad, además del enlace a los contenidos, un resumen o varios resúmenes de los contenidos intercalados en el propio enunciado de la actividad.

# 3. Explicaciones que se repiten

Hay ocasiones en clase, que estamos repitiendo las mismas explicaciones por diversos motivos. Normalmente son aclaraciones de conceptos especialmente dífíciles.

En esos casos, creamos un fichero con las [explicaciones](./../explicaciones) y en las actividades ponemos un enlace a este fichero.

Esta sección puede desaparecer en un futuro. Las explicaciones se irán moviendo poco a poco a sus correspondientes actividades y con el tiempo irán desapareciendo o cambiando a según las dificultades que presenten los alumnos cada nuevo curso.
